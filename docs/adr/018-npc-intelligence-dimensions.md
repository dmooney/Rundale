# ADR-018: NPC Multidimensional Intelligence

## Status

Accepted

## Context

NPC dialogue is generated by LLMs using system prompts that include
personality descriptions, mood, relationships, and knowledge. While
personality text captures *who* a character is, it doesn't explicitly
encode *how smart* they are or *in what ways*. A publican and a priest
may both be "intelligent" but in very different dimensions — the publican
reads people and thinks practically, while the priest reasons abstractly
and speaks eloquently.

Without explicit intelligence signals, the LLM tends to produce
uniformly articulate, thoughtful dialogue regardless of character. A
70-year-old storyteller and a 22-year-old restless dreamer end up
sounding similarly competent. We need the LLM to differentiate:

- A farmer who speaks plainly vs. a teacher who uses rich vocabulary
- A constable who misses emotional cues vs. a publican who reads the room
- A shopkeeper who reasons analytically vs. a storyteller who thinks in
  metaphors

## Decision

Each NPC gets a six-dimension intelligence profile rated 1–5:

| Code | Dimension | What it governs |
|------|-----------|-----------------|
| **V** | Verbal | Vocabulary, eloquence, language complexity |
| **A** | Analytical | Logic, reasoning, problem-solving |
| **E** | Emotional | Empathy, reading subtext, social awareness |
| **P** | Practical | Common sense, hands-on resourcefulness |
| **W** | Wisdom | Life experience, judgment, foresight |
| **C** | Creative | Imagination, wit, metaphor, improvisation |

### Prompt encoding

To minimize token overhead, we inject intelligence as a compact tag
plus a one-time legend:

```
INTELLIGENCE KEY: V=verbal/eloquence A=analytical/logic E=emotional/empathy
P=practical/resourcefulness W=wisdom/judgment C=creative/wit. Scale 1-5
(1=very low, 3=average, 5=exceptional). Match dialogue complexity, vocabulary,
reasoning depth, emotional perception, and wit to these ratings.

INT[V3 A3 E4 P4 W5 C4]
Read subtext and respond to unspoken feelings. Offer concrete, hands-on solutions.
Draw on life experience; give measured counsel. Be witty and inventive; use vivid metaphors.
```

**Token cost**: ~60 tokens for the legend (once per system prompt) + ~20 tokens
for the tag + 0–40 tokens for behavioral hints. Total overhead: ~80–120 tokens
per NPC interaction, well within budget.

### Behavioral guidance

For dimensions rated 4–5 or 1–2, short directive hints are auto-generated
(e.g., "Use rich vocabulary" for V5, "Speak simply; struggle with complex
words" for V1). Dimensions at 3 produce no hints to save tokens.

## Consequences

- **Differentiated dialogue**: Each NPC's speech patterns, reasoning style,
  and emotional awareness now reflect their intelligence profile.
- **Low token cost**: The compact `INT[...]` encoding adds minimal overhead
  compared to verbose prose descriptions of intelligence.
- **Data-driven**: Intelligence ratings live in `npcs.json` alongside
  personality, making them easy to tune per character.
- **Backward compatible**: The `intelligence` field defaults to all-3s if
  omitted from JSON, so existing save files and test data work unchanged.
- **Extensible**: New dimensions can be added later (e.g., Musical,
  Spiritual) by extending the struct and legend.

## Implementation

- `Intelligence` struct in `crates/parish-core/src/npc/types.rs`
- `intelligence` field on `Npc`, `NpcFileEntry`, `NpcSnapshot`
- Injected into Tier 1 system prompts via `prompt_tag()` + `prompt_legend()`
  + `prompt_guidance()`
- Injected into Tier 2 prompts as compact tags per NPC in the character list
- Ratings defined per NPC in `data/npcs.json`
